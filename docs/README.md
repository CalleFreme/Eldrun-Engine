# Documentation folders

## docs/

Stores architecture documents, setup guides, decisions, and onboarding material.

## docs/architecture/

High-level system and module design documents.
Examples:

* engine architecture
* rendering architecture
* future networking considerations
* world/simulation architecture

## docs/guides/

Practical how-to documentation.
Examples:

* Repository bootstrap guide (also found in root README)
* building on Windows
* adding a new game project
* importing assets

## docs/decisions/

Architecture decision records or short design-decision notes.
Examples:

* Why SDL3 was chosen
* Why bgfx was chosen
* WHy fixed-timestep simulation was adopted
